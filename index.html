2.1 Atributos de los sistemas y aplicaciones basados en web

Web-based systems and applications have specific attributes that differentiate them from traditional desktop applications. These attributes are critical for ensuring that web applications meet user needs, are scalable, and can operate effectively in a distributed environment.

2.1.1. Scalability

Scalability refers to a system’s ability to handle increasing amounts of work or its potential to expand to accommodate growth. In web applications, scalability is crucial because the number of users or transactions can grow rapidly. A scalable web application should be able to handle more requests without compromising performance. There are two types of scalability:

  • Vertical Scalability: Increasing the capacity of a single server by adding resources like CPU, memory, or storage.
  • Horizontal Scalability: Adding more servers to distribute the load, often achieved through load balancing.

Real-World Example: Amazon’s e-commerce platform is an excellent example of a scalable web application. As demand increases, Amazon can distribute the load across multiple servers worldwide.

2.1.2 Availability

Availability is the ability of a system to remain accessible and operational at all times. High availability is achieved by minimizing downtime and ensuring that users can access the application whenever needed. Techniques to ensure high availability include redundancy, failover mechanisms, and geographically distributed servers.

  • Redundancy: Keeping backup systems in place so that if one fails, another can take over.
  • Failover: Automatically switching to a backup system when the primary one fails.

Real-World Example: Google’s search engine is designed to be highly available. Even during maintenance or unexpected outages, users can still access the service.

2.1.3 Security

Security is a critical attribute for web applications due to the risk of cyberattacks, data breaches, and unauthorized access. Key security features include:

  • Encryption: Ensuring that data transmitted between the client and server is secure (e.g., using HTTPS).
  • Authentication and Authorization: Verifying user identities and ensuring they have the correct permissions to access resources.
  • Data Protection: Protecting sensitive data like passwords and personal information from being exposed or stolen.

Real-World Example: Online banking platforms like PayPal implement strong encryption protocols and multi-factor authentication to secure user data.

2.1.4 Performance

Performance measures how quickly a web application responds to user requests. Users expect fast load times and smooth interactions, so optimizing performance is crucial for a positive user experience. Key factors affecting performance include:

  • Server Response Time: The time it takes for a server to process a request and return a response.
  • Bandwidth: The amount of data that can be transmitted over the internet in a given time.
  • Caching: Storing copies of frequently accessed data in a cache to reduce server load and improve response times.

Real-World Example: Social media platforms like Instagram and Facebook optimize performance by using content delivery networks (CDNs) to deliver images and videos more quickly.

2.1.5. User Experience (UX)

The user experience is critical for the success of a web application. UX encompasses everything from the visual design to the ease of use and responsiveness of the application. A well-designed web application should be intuitive, fast, and accessible across a range of devices.

  • Responsive Design: Adapting the layout and functionality of a web app to different screen sizes and devices.
  • Accessibility: Ensuring that people with disabilities can use the web application effectively.

Real-World Example: Airbnb’s web application provides a highly intuitive user experience, with responsive design that works seamlessly on mobile devices and desktop computers.
