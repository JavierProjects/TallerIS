2.2 Estratos de la Ingeniería de Webapp

La ingeniería de aplicaciones web se organiza en diferentes estratos o capas, cada una con responsabilidades específicas. Estos estratos garantizan una separación lógica entre la presentación, la lógica de negocio, y la gestión de datos, facilitando así el desarrollo, mantenimiento y escalabilidad del sistema.

1. Estrato de Presentación

El estrato de presentación es la capa visible para los usuarios y se encarga de la interacción entre el sistema y el usuario final. Está compuesto por interfaces gráficas (GUIs) o páginas web que responden a las acciones del usuario.

  • Lenguajes comunes: HTML, CSS, JavaScript.
  • Frameworks: React, Angular, Vue.js.
  • Función principal: Presentar la información y recibir datos del usuario.

2. Estrato de Lógica de Negocio

Este estrato es responsable de procesar la lógica de negocio, que define las reglas, procesos y operaciones que el sistema debe ejecutar para cumplir sus objetivos. Conecta el estrato de presentación con la capa de datos.

  • Lenguajes comunes: Python, Java, JavaScript (Node.js).
  • Frameworks: Django, Spring, Express.
  • Función principal: Ejecutar las reglas de negocio y coordinar las acciones entre capas.

3. Estrato de Gestión de Datos

El estrato de gestión de datos es responsable de almacenar y recuperar la información utilizada por el sistema. Incluye bases de datos relacionales y no relacionales, así como servicios de almacenamiento en la nube.

  • Bases de datos comunes: MySQL, MongoDB, PostgreSQL.
  • Servicios en la nube: Amazon S3, Firebase, Azure.
  • Función principal: Gestionar los datos del sistema de forma segura y eficiente.

4. Estrato de Seguridad

La capa de seguridad atraviesa todos los estratos para proteger la aplicación y los datos del sistema. Incluye la implementación de medidas como la encriptación, autenticación y control de acceso.

  • Protocolos comunes: HTTPS, OAuth, JWT.
  • Función principal: Asegurar la confidencialidad, integridad y disponibilidad de la información.

5. Estrato de Infraestructura

El estrato de infraestructura incluye los servidores, redes y otros recursos físicos o virtuales necesarios para ejecutar la aplicación. También cubre aspectos relacionados con la escalabilidad y la disponibilidad del sistema.

  • Servicios comunes: AWS, Google Cloud, Azure.
  • Función principal: Proveer los recursos necesarios para ejecutar y mantener la aplicación.

Conclusión

Cada estrato de la ingeniería de aplicaciones web desempeña un rol esencial para garantizar que el sistema funcione de manera efectiva. Una clara separación entre las capas facilita la gestión del proyecto y permite que el sistema sea escalable, seguro y fácil de mantener.
